# validator-generator (Skill 3 of 4)

Turns a hand-curated schema YAML into a `validate_<component>_schema(data)` function the framework's tests can import. Assert-based (matches existing `src/validators.py` style), recursive for nested objects/arrays, no external dependency beyond `typing.Any`.

## Pipeline position

```
testcase-generator   →  spec sheet (xlsx)
test-data-generator  →  test_data JSON
validator-generator  →  validate_<component>_schema(data)   ← THIS SKILL
test-generator       →  pytest module
```

## When to invoke

- `/validator-generator`
- "generate validator from schema"
- "build the validate_<component>_schema function"

## Inputs

| Input | Format | Location | Purpose |
|---|---|---|---|
| Component schema | `.yaml` | `received_schemas/<component>.yaml` | Required/optional fields, types, nested shapes, numeric ranges |

### Schema YAML format

```yaml
component: countries                    # required — names the validator
validator_name: validate_countries_schema   # required — function name in output
description: "Validates a single country object returned by REST Countries v3.1"

shape:
  type: object
  required: [name, capital, population, currencies, languages]
  fields:
    name:
      type: object
      required: [common, official]
      fields:
        common:   { type: string }
        official: { type: string }
    capital:    { type: array, item_type: string }
    population: { type: integer, min: 0 }
    currencies: { type: object }
    languages:  { type: object }
    latlng:
      type: array
      item_type: number
      min_length: 2
      max_length: 2
```

### Supported field types

| Type | Python check | Modifiers |
|---|---|---|
| `string` | `isinstance(v, str)` | `non_empty: true` |
| `integer` | `isinstance(v, int)` | `min: N`, `max: N` |
| `number` | `isinstance(v, (int, float))` | `min: N`, `max: N` |
| `boolean` | `isinstance(v, bool)` | — |
| `number_or_null` | `v is None or isinstance(v, (int, float))` | — |
| `string_or_null` | `v is None or isinstance(v, str)` | — |
| `object` | `isinstance(v, dict)` | `required`, `fields` (recursive) |
| `array` | `isinstance(v, list)` | `item_type`, `min_length`, `max_length`, `non_empty` |

## Outputs

| File | Format | Location | Contents |
|---|---|---|---|
| Validator module | `.py` | `generated_validators/validate_<component>_<YYYYMMDD_HHMMSSZ>.py` | One top-level `def validate_<component>_schema(data: dict[str, Any]) -> None:` with recursive checks |

The user reviews the file, then either:
- **Manually pastes** the function into `src/validators.py`, or
- Re-runs with `--write-to-validators` to append it into `src/validators.py` inside a marked auto-gen block.

## CLI

```bash
# Generate validator for every schema in received_schemas/
python .claude/skills/validator-generator/scripts/run.py

# Generate for a single component
python .claude/skills/validator-generator/scripts/run.py --schema countries

# Also append into src/validators.py inside a marked auto-gen block
python .claude/skills/validator-generator/scripts/run.py --schema countries --write-to-validators

# Force overwrite if a function with the same name already exists in the marked block
python .claude/skills/validator-generator/scripts/run.py --schema countries --write-to-validators --force
```

### Flags

- `--schema <name>` — only process `received_schemas/<name>.yaml` (default: all schemas)
- `--write-to-validators` — append the generated function into `src/validators.py` inside a marked block (`# === auto-generated validators (DO NOT EDIT) ===` … `# === end auto-generated ===`)
- `--force` — required to overwrite an existing same-named function inside the auto-gen block
- `--verbose` — log per-field assertion emission

## Output sample

For the countries schema above, the emitted Python:

```python
"""
Auto-generated by validator-generator skill.
Source schema: countries.yaml
Generated:     2026-05-21T...

Move the function below into src/validators.py — or re-run with
--write-to-validators to do it automatically.
"""
from typing import Any


def validate_countries_schema(data: dict[str, Any]) -> None:
    """Validates a single country object returned by REST Countries v3.1."""
    assert isinstance(data, dict), f"root must be dict, got {type(data).__name__}"

    for required in ("name", "capital", "population", "currencies", "languages"):
        assert required in data, f"missing required field: {required}"

    assert isinstance(data["name"], dict), "name must be dict"
    for required in ("common", "official"):
        assert required in data["name"], f"missing: name.{required}"
    assert isinstance(data["name"]["common"], str), "name.common must be str"
    assert isinstance(data["name"]["official"], str), "name.official must be str"

    assert isinstance(data["capital"], list), "capital must be list"
    for _item in data["capital"]:
        assert isinstance(_item, str), "capital items must be str"

    assert isinstance(data["population"], int), "population must be int"
    assert data["population"] >= 0, "population must be >= 0"

    assert isinstance(data["currencies"], dict), "currencies must be dict"
    assert isinstance(data["languages"], dict), "languages must be dict"
```

## Reuse map (zero duplication)

| Reused | From | How |
|---|---|---|
| Validator assert-style | `src/validators.py` existing functions | pattern only — no code import |

No HTTP, no spec-sheet read, no test_data — this skill is purely a schema-to-Python emitter.

## Logging & exception handling

- `LOG = logging.getLogger(__name__)` in every script.
- INFO logs: "Loaded schema X", "Emitted N assertions", "Wrote validator file Y".
- WARNING: unsupported type encountered → skipped with reason.
- All errors raised as `SchemaError` and caught at `run.py` boundary → `[validator-generator] <message>` on stderr, exit 2.
- `--write-to-validators` refuses to overwrite without `--force`, refuses if existing function lives outside the auto-gen block.

## Verification

After running on `received_schemas/countries.yaml`:

1. `generated_validators/validate_countries_<ts>.py` parses cleanly via `ast.parse`.
2. `flake8 --select=E9,F` on the file reports no errors.
3. `python -c "from generated_validators.validate_countries_<ts> import validate_countries_schema; validate_countries_schema({...valid object...})"` → no exception.
4. `python -c "from ... import validate_countries_schema; validate_countries_schema({})"` → `AssertionError: missing required field: name`.
